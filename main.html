2. What is the difference between innerHTML, innerText, and textContent ?

Answer

Great question! Let's break down the differences between innerHTML, innerText, and textContent

innerHTML :
innerHTML is a property in the Document Object Model (DOM) that allows you to get or set the HTML content inside an element..
1. It's more performance-intensive because it causes the browser to reparse and re-render the entire HTML structure.
2. Useful for retrieving or changing the HTML structure, including tags and attributes.
3. It handles the full HTML content, including tags and their attributes.

innerText :
innerText is a property in JavaScript that retrieves or sets the visible text content of an element, considering CSS styles
1.It considers CSS styles when retrieving text.
2. It can be used to change the text content of an element.
3.It triggers layout reflow, making it slower than textContent.

textContent :
textContent is a property that retrieves or sets the raw text inside an element, including hidden text, without considering CSS styles.

3. What is event delegation in the DOM?

Answer

Event delegation is a powerful technique in the DOM that involves using a single event listener to handle events for multiple elements. where an event propagates from the element to the outermost elements.

1. Event Bubble: When an event is on an element, it triangulates on the element between it and then bubbles up to its parent element, and so on, not grouping it into the parent element.
2. You make an event listener with a common ancestor element. This reduces memory usage and improves performance, especially when working with many components.

4. What is event bubbling in the DOM?

Answer

Event bubbling is a concept in the Document Object Model (DOM) where an event triggered on a specific element "bubbles up" to its parent elements and eventually to the root of the document. This means that an event can be captured and handled at multiple levels of the DOM hierarchy.

When an event occurs on an element, it first triggers on that element (the target element). Then, it propagates up through its parent elements (ancestor elements) until it reaches the root element the document object.

1. Event Target Phase: The event is triggered on the target element the button.
2. Event Bubbling Phase: The event bubbles up to the parent element (the div) and continues up to the root element document.

5. How do you create, add, and remove elements using JavaScript?

Answer

creating, adding, and removing elements in JavaScript involves generating the element, inserting it into the desired position in the DOM

1. Creating Elements: create a new element, you use the document.createElement() method, which allows you to generate any HTML element you need.

Adding Elements: Once you've created an element, you can add it to the DOM using methods like appendChild(), add() or append(). You'll typically specify a parent element to which the new element will be added.

Removing Elements: The remove an element from the DOM, you use the removeChild() method or the remove() method. With removeChild(), you need to specify the parent element from which you're removing the child element. The remove() method directly deletes the element from the DOM without needing to specify the parent.
